# PRODUCT_CONSTITUTION

## Product
SwingCue is a consumer AI golf swing correction app for beginners and early intermediate golfers.

## Value promise
Upload your swing. See what’s wrong. Know what to fix next.

## MVP boundaries
SwingCue MVP is:
- mobile-first
- consumer-grade
- minimal
- focused on visual correction guidance

SwingCue MVP is not:
- a coach marketplace
- a launch monitor platform
- a pro biomechanics workstation
- a social product
- a comparison workspace
- a dashboard product

## Core MVP rules
1. The system gives one primary problem at a time.
2. Result page is visual-first.
3. Output language must be plain English and beginner-friendly.
4. MVP supports only two views:
   - face_on
   - down_the_line
5. The system must use explicit states, not inferred states.

## Main user journey
1. User signs in.
2. User uploads a swing video.
3. User selects a supported view.
4. System stores the upload and creates a persistent record.
5. System enters processing.
6. Deterministic stub completes analysis.
7. User sees a result page.
8. User can re-open past results from history.

## One-One-One rule
For a completed analysis, the user sees:
- one main issue
- one fix cue
- one drill suggestion

## Simplicity rules
Do not introduce:
- multi-issue ranking
- advanced charts
- dense metrics dashboards
- pro-style terminology
- excessive forms
- feature sprawl

## Success test for MVP
A first-time user should be able to upload one swing, understand one problem, and know the next action without needing a coach or a manual.
